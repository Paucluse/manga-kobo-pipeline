## Major Code Changes In Current Dirty Worktree

The worktree is intentionally dirty and should not be reverted.

- BookWalker Taiwan provider:
  - Converts queries to Taiwan traditional Chinese.
  - Tries volume query variants like `2`, `02`, and no-volume fallback.
  - Cleans BookWalker control-character artifacts.
  - Improved matching, but still needs stricter false-positive handling in future work.

- Gemini LLM filename normalization:
  - OpenAI-compatible endpoint support.
  - Configured for Gemini:
    - `llm_base_url: https://generativelanguage.googleapis.com/v1beta/openai`
    - `llm_model: gemini-3.1-flash-lite`
    - `llm_api_key_file: /run/secrets/gemini_api_key`
  - Key file is mounted from `./secrets/gemini_api_key`.

- Collection directory handling:
  - Top-level `inbox/<series>/` is treated as a collection.
  - Only one directory level is processed.
  - Child archives/PDFs are treated as books.
  - Child image directories are packed as single books.
  - Collection directory title is now a strong series constraint.
  - Examples:
    - `三只眼 典藏版 [高田裕三][玉皇朝]` forces series to `三隻眼 典藏版`.
    - `[Dr.SLUMP怪博士與機器娃娃][鳥山明][東立][nearfly][15完]` uses the first three bracket groups as title/author/publisher.

- PDF handling:
  - `.pdf` is supported.
  - Default strategy is `pdfimages` extraction first, not high-DPI render.
  - Original PDF can be preserved as `.source.pdf`.

- KCC safety:
  - Removes stale exact outputs before conversion.
  - Rejects KEPUB outputs whose image count is lower than source CBZ.
  - Prevents importing partial/stale KCC outputs.

- Rescrape command:
  - `manga-pipeline rescrape`
  - Supports `--id`, `--title`, `--all`, `--dry-run`, `--relocate`, and related options.

- NAS script:
  - `scripts/` contains NAS mount helper work. Review before relying on it.
